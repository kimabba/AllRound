변경 이유 구조 디자인 보안 후속 논의 인수인계

2026.07.19 · Product & Design Handoff

올라운드를
오늘 쓰는 앱으로.

기능을 나열하던 첫 화면을 사용자의 오늘과 다음 행동이 보이는 화면으로 바꾸고, AI 채팅은 특정 메뉴가 아니라 어디서나 이어지는 공통 도구로 재배치했습니다. 이 문서는 kimabba, 백과장, 시리가 같은 이유와 기준을 보고 다음 작업을 이어가기 위한 인수인계서입니다.

이번 결정

백과장 사전 검수를 머지 조건으로 두지 않고 현재 구조를 메인에 반영합니다. 실제 기기와 운영 환경에서 발견되는 개선점은 이 기준을 출발점으로 별도 논의합니다.

기준 디자인 Pureform Sports
기본 내비게이션 4탭 + 전역 AI
검증 데이터 로컬 합성 계정만 사용
기준 브랜치 codex/global-chat-sheet
01 · WHY TODAY

왜 홈을
‘오늘’로 바꿨나

‘홈’은 위치 이름이고 ‘오늘’은 사용 목적입니다. 앱을 열었을 때 사용자가 설명을 읽는 대신 지금 확인할 일정과 바로 할 행동을 고르게 만들었습니다.

01

기능보다 상황을 먼저

메뉴 모음이 아니라 날짜, 맞춤 질문, 추천 대회를 먼저 보여줘 첫 화면의 판단 시간을 줄였습니다.

02

다음 행동이 바로 보이게

추천, 이번 주, 전체 필터와 대회 목록을 같은 흐름에 두어 탐색을 시작하기 쉽게 했습니다.

03

AI를 메뉴 밖으로

대부분의 정보 탐색이 질문으로 이어지므로 채팅을 별도 탭에서 빼고 모든 주요 화면 아래에 두었습니다.

02 · INFORMATION ARCHITECTURE

사용 구조가
이렇게 달라졌다

탭 수를 줄인 것이 핵심이 아닙니다. 사용자가 ‘어디로 가야 하지?’를 덜 고민하도록 운동 탐색, 커뮤니티, 개인 관리, 질문이라는 네 가지 목적을 분명히 나눈 변화입니다.

BEFORE · 기능 중심
  • 라운드 코치첫 화면의 정체성이 모호함
  • 대회목록 탐색
  • 클럽커뮤니티 탐색
  • 코치 탭채팅을 위해 이동
  • 더보기룰북과 설정 분산

  • 클럽검색, 가입, 운영
  • MY프로필, 기록, 관심, 설정
↗ AI에게 물어보기 · 모든 주요 화면에서 바로 연결
STEP 1

현재 화면에서 질문

오늘, 대회, 클럽, MY 어디서나 하단 AI 진입창을 누릅니다.

STEP 2

반 높이 시트로 시작

현재 화면을 잃지 않고 추천 질문을 보거나 바로 입력합니다.

STEP 3

문맥은 직접 연결

대회와 클럽 정보는 사용자가 선택한 경우에만 질문에 포함합니다.

STEP 4

필요하면 전체 화면

작성 중인 내용을 유지한 채 전체 채팅으로 확장합니다.

03 · DESIGN SYSTEM

한 화면이 아니라
전체 톤을 바꿨다

Apple Sports처럼 정보를 먼저 읽히게 하되 그대로 복제하지 않았습니다. 앱 전체가 같은 제품처럼 느껴지도록 색, 여백, 글자, 모서리, 상태 표현을 한 기준으로 묶었습니다.

오늘 /
  • 오늘 날짜와 짧은 질문을 첫 위계로 배치
  • 추천, 이번 주, 전체 대회를 평면 목록으로 탐색
  • 중복 AI 버튼을 없애고 전역 하단 채팅만 유지
대회 /tournaments
  • 날짜, 대회명, 장소, 상태의 읽기 순서를 통일
  • 목록, 상세, 신청을 같은 여백과 구분선으로 정리
  • 대회 문맥은 AI 연결을 직접 켠 경우에만 사용
클럽 /clubs
  • 로고, 이름, 지역, 인원, 관심 행동만 우선 노출
  • 가입 신청자와 운영자의 권한별 화면을 분리
  • 상세와 생성 화면도 같은 평면 섹션으로 정리
MY /profile
  • 프로필, 기록, 종목, 알림, 계정 설정을 한 흐름으로 묶음
  • 룰북, 관심, 차단 관리를 개인 메뉴에서 연결
  • 친구 일정 콘셉트는 데이터 기준 확정 전 출시 경로에서 제외
색 오프화이트 표면, 짙은 텍스트, 코발트 한 가지 포인트 종목마다 다른 강조색을 쓰지 않아 화면 경쟁을 줄임
정보 구분 큰 카드와 그림자 대신 여백과 1px 구분선 사용 장식보다 날짜, 제목, 상태가 먼저 읽힘
간격 좌우 20px, 섹션 24px 또는 32px, 8px 기반 리듬 화면마다 달랐던 시작선과 밀도를 통일
조작 영역 버튼과 주요 터치 영역을 원칙적으로 48px 이상 확보 모바일 조작 실수와 접근성 문제를 줄임
상태 로딩, 결과 없음, 오류, 재시도, 권한 제한을 같은 자리에서 표현 문제가 생겨도 사용자가 다음 행동을 알 수 있음
반응형 390px 모바일 기준, 320px와 큰 글자 조건까지 자동 검사 웹에서도 모바일 앱 폭과 정보 위계를 유지
04 · SECURITY & PRIVACY

디자인과 함께
안전 경계도 보강

화면이 좋아져도 계정 사이의 데이터가 섞이거나 삭제가 불완전하면 출시할 수 없습니다. 이번 변경은 UI뿐 아니라 가입, 채팅, 이미지, 푸시, 캐시, 탈퇴 경계를 함께 다룹니다.

ACCOUNT

가입 전에 연령 확인

이메일 가입은 생년월일을 먼저 받고, 만 14세 미만이나 잘못된 값은 계정이 생기기 전에 서버에서 차단합니다.

CHAT

계정별 대화와 문맥 분리

로그인 사용자가 바뀌면 대화, 초안, 대화 ID를 초기화하고, 앱이 AI 답변을 임의로 위조해 저장하지 못하게 보강했습니다.

MEDIA

사진 메타데이터 제거

업로드 전에 이미지를 다시 인코딩해 촬영 위치, 시간, 코멘트 같은 EXIF와 PNG 텍스트 정보를 제거합니다.

OWNERSHIP

푸시와 저장소 소유권 강화

물리 푸시 토큰은 한 계정에만 속하게 하고, 파일 목록과 쓰기는 로그인 토큰의 소유자 기준으로 검사합니다.

DELETE

탈퇴를 다시 시도할 수 있게

일부 단계가 실패해도 탈퇴를 안전하게 재시도하고, 공개 사진과 계정 데이터를 순서대로 정리하도록 바꿨습니다.

QA

실제 회원 대신 합성 계정

자동 테스트는 로컬 Supabase와 전달 불가능한 테스트 이메일만 사용합니다. 운영 DB, 실제 개인정보, 실제 알림은 사용하지 않습니다.

05 · RELEASE STATUS

머지와 출시는
같은 뜻이 아니다

로컬 자동 검사는 넓게 통과했지만, 메인 머지는 코드 기준선을 정하는 단계입니다. 운영 Supabase 설정, 실제 기기, 법무 문구는 별도 출시 게이트로 남습니다.

이번 변경에서 확인한 것

  • Flutter 분석과 위젯 테스트
  • Deno 포맷, 린트, 타입, 함수 테스트
  • 로컬 DB 초기화와 권한 회귀 테스트
  • 8개 합성 계정의 주요 역할별 흐름
  • Chrome과 macOS 앱의 핵심 E2E
  • 18개 모바일 화면 증거와 접근성 기준
  • 비밀키 패턴과 테스트 산출물 유출 검사

출시 전에 별도 확인할 것

  • 운영 Auth Hook, RLS, Edge Function 설정 일치
  • 실제 iOS와 Android의 큰 글자, VoiceOver, TalkBack
  • 실제 OAuth, 카메라, 알림 권한, FCM 로그아웃
  • AI 제공자 안내, 보관 기간, 삭제 방식의 정책 문구
  • 신고 증거 이미지의 보존 기간과 자동 만료
  • 실제 모델을 격리한 프롬프트 공격 평가
  • iOS 시뮬레이터를 막는 ffmpeg 플러그인 대응
머지 후 UI 우선 점검: 200% 글자 크기에서 하단 AI 진입창과 MY 일부 고정 높이 영역, 달력의 작은 날짜 조작 영역, 관리자 대회 편집의 저장 버튼 가시성, 대회 장소가 없을 때 지역을 장소처럼 보여주는 대체 문구를 실제 화면에서 다시 확인합니다. 이는 이번 머지를 되돌리는 조건이 아니라 후속 개선 목록입니다.
06 · PEOPLE & HANDOFF

누가 무엇을
기준으로 볼 것인가

같은 화면을 보고도 제품, 디자인, 구현 관점은 다릅니다. 이번 문서는 승인 절차를 늘리기보다 서로 다른 역할이 같은 출발점에서 후속 논의를 할 수 있게 만드는 기준입니다.

KIMABBA · PRODUCT

사용 목적과 우선순위

오늘 화면이 실제로 쓸모 있는지, 하단 AI가 핵심 기능처럼 느껴지는지, 다음 출시 범위를 결정합니다.

  • 구조 변경 최종 판단
  • 운영·법무 후속 결정
  • 실제 사용자 불편 우선순위
백과장 · DESIGN REVIEW

머지 후 화면 품질 검수

사전 승인자가 아니라 후속 개선 파트너로 봅니다. 실제 기기의 여백, 글자 위계, 조작 크기, 화면 간 일관성을 확인합니다.

  • 실제 기기 시각 검수
  • Pureform 기준 이탈 표시
  • 개선안을 화면 단위로 제안
시리 · IMPLEMENTATION

현재 구조를 보존하며 구현

새 기능을 붙일 때 4탭과 전역 AI 구조를 임의로 되돌리지 않습니다. 공통 토큰과 위젯을 먼저 사용하고 구조 변경은 다시 설명 후 확인받습니다.

  • 클럽과 디자인 레인 우선
  • 공통 파일 충돌 전 조율
  • 로딩·빈 상태·오류까지 구현
