# 올라운드 디자인 시스템 — Pureform Sports

> 확정: 2026-07-18 (kimabba). `Active Bold`를 대체하는 현재 앱 디자인 기준이다.
> 참고 방향은 Apple Sports의 정보 우선순위와 Cal.com 계열의 명료한 인터랙션이며, 특정 제품을 복제하지 않는다.

## 1. 한 문장 원칙

운동 정보를 가장 빠르게 읽고 다음 행동을 바로 고를 수 있는, 차갑고 단정한 스포츠 도구.

## 2. 시각 언어

| 역할 | 값 | 사용 |
|---|---|---|
| Canvas | `#F4F5F7` | 앱 배경 |
| Surface | `#FCFCFB` | 목록·패널 |
| Ink | `#161616` | 제목·핵심 숫자 |
| Muted | `#6F7176` | 보조 정보 |
| Border | `#E5E6E8` | 구분선·외곽선 |
| Cobalt | `#3156D8` | 선택·링크·주요 행동 하나 |
| Cobalt tint | `#EDF1FF` | 선택 배경 |

- 종목마다 경쟁하는 강조색을 만들지 않는다. 테니스와 풋살도 같은 코발트 계열을 사용한다.
- 그라데이션, 장식용 그림자, 유리 효과, 크고 둥근 카드, 의미 없는 배지를 사용하지 않는다.
- 정보 그룹은 카드보다 여백과 1px 구분선으로 나눈다.

## 3. 형태와 간격

- 화면 좌우 여백: 20px
- 최소 터치 영역: 44px
- 버튼: 높이 48px, radius 6px
- 목록·패널: radius 8px, 꼭 필요한 경우에만 1px border
- 큰 패널 최대 radius: 12px
- 그림자: 기본값 없음. 모달 오버레이만 사용

## 4. 타이포와 정보 위계

- Pretendard 유지
- 화면 제목과 핵심 숫자는 굵게, 자간을 조금 좁힌다.
- 대회 목록은 날짜 숫자를 가장 먼저 읽히게 하고 제목, 장소, 상태 순으로 배치한다.
- 본문은 한 화면에서 같은 굵기를 반복하지 않는다. `제목 → 핵심 메타 → 보조 메타` 3단계로 제한한다.

## 5. 내비게이션

텍스트 중심 5탭을 사용한다.

| 탭 | 경로 | 역할 |
|---|---|---|
| 오늘 | `/` | 맞춤 일정과 빠른 행동 |
| 대회 | `/tournaments` | 검색·달력·상세·신청 |
| 클럽 | `/clubs` | 검색·가입·운영 |
| 코치 | `/chat` | 어디서나 바로 묻는 AI 코치 |
| MY | `/profile` | 프로필·기록·설정 진입 |

룰북은 MY의 도움말 메뉴에서 `/rules`로 진입한다. 알림과 설정도 각 화면의 문맥상 액션으로 둔다.

## 6. 화면별 기준

- 홈: 날짜, 한 문장 질문, 추천 대회 목록을 첫 화면에 둔다.
- 대회: 큰 날짜 숫자와 평면 목록을 기본으로 하고 달력은 보조 탐색으로 둔다.
- 클럽: 로고·이름·지역·인원·관심 행동만 한 행에 남긴다.
- 룰북: 카테고리와 글을 문서 목록처럼 보이게 한다.
- MY: 프로필과 핵심 통계는 한 덩어리로, 세부 메뉴는 구분선 목록으로 둔다.
- 인증·온보딩: 제품 설명보다 입력과 다음 행동이 먼저 보이게 한다.
- 알림·관심·차단: 별도 장식 카드 대신 평면 목록, 구분선, 명확한 빈 상태를 사용한다.
- 입력 화면: 섹션 제목과 필드 순서로 위계를 만들고 카드 중첩을 피한다.

## 7. 구현 규칙

- 색과 radius는 `app/lib/theme/` 토큰을 사용한다.
- 공통 표면은 `AppCard`의 flat/outlined 변형을 우선한다.
- 기능 상태를 색만으로 전달하지 않는다. 텍스트·아이콘·형태를 함께 사용한다.
- 로딩, 오류, 빈 상태에서도 레이아웃 위계가 바뀌지 않게 한다.
- 새 UI는 라이트·다크 테마, 큰 글자, 44px 터치 영역을 함께 확인한다.

## 8. 디자인 프리뷰

`USER_DESIGN_PREVIEW=true`인 웹 실행은 운영 인증·권한을 대신하지 않는 로컬 UI 점검 전용이다.

- 데스크톱 브라우저에서도 앱 본문을 최대 390px 폭으로 고정해 모바일 레이아웃을 확인한다.
- `/login`, `/onboarding`, `/notifications`, `/favorites`, `/blocked-users`, `/rules`, `/speed-gun` 등 원하는 사용자 경로를 주소에서 바로 열 수 있다.
- 홈·대회·클럽·룰북·알림·관심·차단 화면은 프리뷰 데이터를 사용해 백엔드 장애와 무관하게 상태별 디자인을 확인한다.
- 릴리스 빌드에서는 기존 `AppConfig.assertConfigured()` 가 프리뷰 플래그 유입을 차단한다.
