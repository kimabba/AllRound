- **법적 검토 완료**: `docs/legal/public-results-usage-review.md`. 진행 가능하나 아래 방어책은 필수.

## 엔드포인트 (실측)

```
목록  https://gjtennis.kr/sub4_5.php?member_kind=<부서>
      → 순위 · 부서 · 사진 · 성명 · 소속 · 순위포인트 · 전체포인트
      → 페이지네이션 없음. 부서당 1요청으로 전체.
상세  https://gjtennis.kr/sub4_5_rank.php?member_kind=<부서>&userid=<id>
      → 대회명 · 순위(우승/4강/32강) · 부서 · 포인트 · 대회일
      → 선수 목록의 player_rank('<id>') 에서 userid 추출
```

## 크롤 설계 (확정)

| 데이터 | 방식 | 이유 |
|---|---|---|
| 랭킹 목록 | 주 1회 배치, **6요청** | 목록 화면에 필요. 부하 거의 없음 |
| 대회 이력(상세) | **열람 시 on-demand + 캐시** | 전량이면 5천 요청. 저작권법 §93② "개별 소재" 범위에 머무는 편이 유리 |
| 사진 | **수집 안 함** | 초상권 별개 축, 값어치 없음 |

- User-Agent 에 신원·연락처 명시 (`AllRoundBot/1.0 (+<도메인>; <이메일>)`)
- 요청 간격 1초 이상, 동시요청 1, 야간 회피
- 협회 이용약관·robots.txt 주기적 재확인 (금지 조항 생기면 즉시 중단·재검토)
