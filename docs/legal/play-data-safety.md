# Google Play 데이터 안전(Data Safety) 답변 시트 — 올라운드

> Play Console "앱 콘텐츠 → 데이터 안전" 폼 작성용 초안. 실제 제출은 kimabba.
> 원칙: Supabase/Google/Firebase는 **우리 위탁 처리자(service provider)** 이므로 대부분 "제3자 공유(shared)=아니오". 단 항목별로 아래 주석 확인.
> ⚠️ 표시 = kimabba 확인/결정 필요.

## 요약 토글
- 데이터를 수집·전송하나요? → **예**
- 전송 시 암호화? → **예** (TLS)
- 사용자가 데이터 삭제를 요청할 수 있나요? → **예** (인앱 회원 탈퇴, JY-112)

## 수집 데이터 항목별 답변

| 데이터 유형 | 수집 | 공유(제3자) | 필수/선택 | 목적 | 비고 |
|---|---|---|---|---|---|
| 이메일 주소 | 예 | 아니오 | 필수 | 계정 관리, 로그인 | Supabase Auth(위탁) |
| 이름 | 예 | 아니오 | 선택 | 계정 관리 | Google 로그인 시만 |
| 프로필 사진(URL) | ⚠️ | 아니오 | 선택 | 프로필 표시 | Google 계정 사진 사용 여부 확인 |
| 사용자 ID | 예 | 아니오 | 필수 | 계정 식별 | Supabase user id |
| 앱 내 활동(종목/등급/지역 설정) | 예 | 아니오 | 선택 | 맞춤 대회 추천 | 사용자 입력값 |
| 앱 내 메시지(채팅) | 예 | ⚠️ 아래 | 선택 | AI 코치 응답 생성 | **Google Gemini로 전송** — 아래 주석 |
| 기타 사용자 콘텐츠(클럽 게시글/댓글) | 예 | 아니오 | 선택 | 커뮤니티 기능 | 앱 내 다른 사용자에게 표시(공유 아님) |
| 기기 식별자(FCM 토큰) | 예 | 아니오 | 선택 | 푸시 알림 | Firebase(위탁) |

## 주석 (중요)
- **위치**: 정밀/대략 위치(GPS) **수집 안 함**. "지역"은 사용자가 선택하는 설정값이므로 위치 데이터로 신고하지 않음(앱 활동/사용자 입력).
- **채팅 메시지 → Gemini**: 채팅 답변 생성을 위해 메시지를 Google Gemini API로 전송. Play "공유(shared)"는 보통 **처리자(service provider)** 로의 전송을 제외하므로 기본은 **공유=아니오 + 수집=예(처리 목적)** 로 답할 수 있음. ⚠️ 단, Google을 처리자로 볼지 kimabba가 최종 판단(계약상 처리자 관계 기준).
- **광고/분석 SDK**: 서드파티 광고·분석 SDK **없음** → 광고·분석 목적 신고 없음. (내부 Gemini 사용량 집계는 개인정보 아님)
- **금융/건강/연락처/사진첩**: 수집 안 함.
- **삭제 요청 경로**: 인앱 회원 탈퇴(JY-112) + 웹 안내 URL(제출 필수). 개인정보는 삭제, 작성 콘텐츠는 익명화 후 보존.

## Apple App Privacy(영양성분표) 매핑 (참고)
- Data Used to Track You: **없음** (트래킹 SDK 없음)
- Data Linked to You: 이메일, 이름, 사용자 ID, 사용자 콘텐츠(채팅·게시글), 기기 ID(FCM)
- Data Not Linked to You: 없음(대부분 계정에 연결)
