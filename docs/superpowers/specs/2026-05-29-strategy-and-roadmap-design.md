## 1. 제품 정의

### 한 줄 정의

**"전남/광주 테니스의 모든 것"** -- 대회 정보, 클럽 관리, AI 챗봇을 하나로. 풋살도 함께.

### 핵심 가치

테니스(풋살) 동호인이 대회 정보를 찾고, 클럽을 운영하고, 궁금한 걸 물어보는 **원스톱 플랫폼**.
앱 하나만 열면 흩어진 정보(협회 홈페이지, 네이버 카페, 밴드, 카톡)를 더 이상 뒤질 필요가 없다.

### 아하 모먼트

> "앱 열었더니 내 등급에 맞는 대회가 바로 보이고, 마감 전에 알려준다."

세 가지 축:
1. **내 등급 대회 자동 필터링** -- 온보딩에서 협회/등급 설정 즉시 적용
2. **AI 챗봇** -- "이번 주 대회 뭐 있어?" 물어보면 즉답
3. **푸시 알림** -- 관심 대회 마감 D-3 자동 알림

## 2. 타겟 사용자

### 1차 타겟: 전남/광주 테니스 동호인

- 광주광역시 테니스협회 130개 동호회, 약 15,000명
- 전남 테니스협회 소속 동호회
- 현재 정보 채널: 협회 홈페이지(그누보드), 네이버 카페, 카카오 오픈채팅, 밴드
- 핵심 페인포인트: 대회 정보 분산, 등급별 필터링 불가, 마감일 놓침

### 2차 타겟: 풋살 동호인

- 대회 정보가 더 심하게 분산 (한국풋살연맹, 네이버 카페, 인스타, 당근마켓 등)
- 통합 대회 정보 플랫폼 부재
- MVP 전략: 풋살은 **인프라 준비 + 기본 UI** 수준. 대회 데이터는 수기 등록 중심.
- Phase 2에서 크롤링 소스 확보 후 본격 확장

| **베이스라인** | - | 대회 검색/신청, 3,360개 시설 | 클럽 관리 약함, AI 없음 |
| **테친소** | 5,000건+ | 무료 매칭, 200배 성장 | 대회/클럽 기능 없음 |
| **KTR** | - | 매치 기반 Elo 레이팅 | 레이팅만, 대회/클럽 없음 |

| 앱 | 사용자 | 강점 | 약점 (Match-up 기회) |
|---|---|---|---|
| **플랩풋볼** | 60만 MAU | 소셜 매치, 400개 구장 | 대회 정보 통합 안 함 |
| **아이엠그라운드** | 30만 | 구장 예약, 팀매칭 | 대회 정보 없음 |

### Match-up 차별점

1. **지역 밀착** -- 수도권 앱이 커버 못하는 전남/광주 데이터
2. **대회 정보 크롤링** -- 협회 사이트 자동 수집 + 등급 매칭
3. **AI 챗봇** -- 기존 앱 중 챗봇 인터페이스를 핵심으로 쓰는 곳 없음
4. **테니스+풋살 통합** -- 두 종목을 함께 다루는 앱 없음

## 4. 전략 로드맵

### Phase 1: MVP (~ 2026-07-22)

> "기본기를 확실히. 대회 정보 + 클럽 + AI 챗봇"

### Phase 2: A2UI + 데이터 확장 (2026 Q3)

> "챗봇이 UI를 그린다. 풋살 크롤링 소스 확보."

- Flutter GenUI SDK 도입 (A2UI 프로토콜)
  - 전제: GenUI SDK 안정 버전 출시 + MVP 챗봇 사용률 DAU 50% 이상 확인 후
- 챗봇 응답에 대회 카드, 클럽 카드 인라인 렌더링
- 풋살 대회 크롤링 소스 확보 및 파서 개발
- 사용자 피드백 기반 기능 개선

### Phase 3: 커뮤니티 + 고도화 (2026 Q4)

> "사용자끼리 연결. 복식 파트너, 레이팅."

- 복식 파트너 매칭
- 레이팅 시스템 (KTR 연동 또는 자체)
- 다크 모드
- 타 지역 확장 검토

### Phase 4: BM + 스케일 (2027~)

> "PMF 확인 후 수익화"

- BM 도입 (광고, 프리미엄 구독, 대회 등록 수수료 등)
- 타 지역/타 종목 확장
- 코트 예약 연동
